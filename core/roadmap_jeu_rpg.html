FEUILLE DE ROUTE — RPG 2D
SAO-like · Vue du dessus · Java · Pixel Art · 2-3 développeurs · 3-6 mois
Java LibGDX recommandé Pixel Art Co-op local
PHASE 1 🏗 Fondations & Prototype Semaines 1–3 ▶

Mettre en place le socle technique avant de toucher au gameplay. C'est la phase la plus critique — une mauvaise architecture ici coûtera des semaines plus tard.

SETUPProjet LibGDX, structure des packages, Git + branches par feature
MOTEURGame loop, delta time, caméra qui suit le joueur, rendu de tiles
CARTETilemap (Tiled), chargement TMX, collisions AABB basiques
JOUEURDéplacement 4 directions, spritesheet animée, hitbox
ASSETSTileset placeholder (16×16 ou 32×32), décider la résolution définitive
CO-OPArchitecture pour 2 joueurs locaux (même clavier ou manettes)
🎯Milestone : un joueur se déplace sur une map, la caméra le suit, les murs bloquent.
PHASE 2 ⚔ Combat & Systèmes RPG Core Semaines 4–7 ▶

Le cœur du gameplay. Tout le reste (boss, artefacts, quêtes) s'appuie sur cette couche. Prendre le temps de la faire proprement.

COMBATAttaque mêlée, cooldown, hitbox d'attaque, knockback
STATSHP, ATK, DEF, SPD, CRIT_RATE, CRIT_DMG (structure Entity)
MOBSIA basique (patrouille → aggro → attaque), drops aléatoires
XP & NIVEAUXCourbe d'XP, level-up avec bonus de stats, affichage
INVENTAIREGrille d'items, équipement (arme, armure, accessoires), tooltips
ITEMSClasse Item, rareté (commun→légendaire), système de tags
ARTEFACTSObjets équipables avec stats bonus, slot dédié, effets passifs
CRAFTRecettes simples : ressource × N → item, interface de craft
🎯Milestone : tuer un mob, looter une épée, l'équiper, voir ses dégâts changer.
PHASE 3 🗺 Monde, Biomes & Donjon Boss Semaines 8–13 ▶

La boucle de progression principale du jeu. Chaque biome est un niveau avec sa grotte, son boss, sa palette visuelle et ses mécaniques propres.

BIOME 1Forêt (carte overworld + grotte + boss — ex: Gardien de Racines)
BIOME 2Désert / Ruines (palette, mobs exclusifs, boss différent)
BOSS IAPhases de combat, patterns d'attaque, seuil de rage à 50% HP
GATE SYSTEMVérifier conditions (items requis, niveau min) avant d'entrer dans la grotte
MINI-DONJONSSalles procédurales optionnelles sur la carte, loot rare
TRANSITIONSPortail vers le biome suivant après kill boss, cinématique simple
FARM LOOPRespawn des mobs, zones de farm balisées, drop rates équilibrés
CARTE UIMini-map, marqueurs de donjons, zone explorée en brouillard
🎯Milestone : farmer les 100 items requis, entrer dans la grotte, battre le boss, changer de biome.
PHASE 4 📜 Quêtes, Histoire & Social Semaines 14–17 ▶

Donner du sens à la progression. Les quêtes guident le joueur, le scénario l'accroche, la guilde lui donne un but long terme.

QUÊTESSystème de quêtes (kill X, collecter Y, explorer Z), journal UI
NPCDialogues simples, marchands, PNJ de quête
SCÉNARIOArc narratif principal : pourquoi les boss existent, lore par biome
GUILDECréer/rejoindre une guilde, objectifs collectifs, classement local
CO-OP LOCAL2 joueurs sur le même écran (split ou shared cam), inventaires séparés
SAUVEGARDESave/load JSON ou SQLite, profils de personnage
🎯Milestone : jouer à 2, suivre une quête principale, sauvegarder et reprendre.
PHASE 5 ✨ Polish & Équilibrage Semaines 18–22 ▶

Ce qui transforme un projet en vrai jeu. Sons, effets, feedbacks visuels, équilibrage des stats — souvent sous-estimé, toujours essentiel.

AUDIOSons d'attaque, ambiance biome, musique de boss (assets libres de droits)
VFXParticules de hit, flash de dégâts, aura des artefacts légendaires
UI/UXHUD propre, écrans de menu, transitions, icônes d'items finales
BALANCECourbes de dégâts, taux de drop, difficulté des boss, test interne
BUGSPlaytests réguliers, tracker d'issues GitHub, correctifs
BUILDPackaging JAR / installeur, README, documentation minimale
🎯Milestone : v0.1 jouable de bout en bout, partageable à des testeurs externes.
💡 RECOMMANDATIONS TECHNIQUES
Utilise LibGDX — parfait pour Java 2D, gère le rendu, l'audio, les inputs manettes, et exporte sur desktop/Android plus tard.
Éditeur de map : Tiled (gratuit) + plugin LibGDX TMX loader. Gain de temps énorme sur la création des biomes.
Organise ton code en ECS léger (Entity/Component/System) dès le départ — ça rend les artefacts et les stats modulaires sans spaghetti.
Pour le co-op local, prévoir 2 gamepads via LibGDX Controllers plutôt que le clavier partagé — bien plus agréable à jouer.
Assets pixel art gratuits de qualité : Itch.io (0-cost assets) pour prototyper vite avant de créer les vôtres.
